### PHASE A: Create the App ID (Apple Developer Portal — ONE TIME)

1. Open https://developer.apple.com/account/resources/identifiers/list
2. Click the **+** button → select **App IDs** → Continue
3. Type: **App** → Continue
4. Fill in:
   - **Description**: `MAK Makeup Buddy`
   - **Bundle ID**: select **Explicit** → enter `com.mak.makeupbuddy`
   - **Capabilities**: leave defaults (no special caps needed)
5. Click **Continue** → **Register**

---

### PHASE B: Create the App in App Store Connect

1. Open https://appstoreconnect.apple.com → **My Apps** → **+** → **New App**
2. Fill in:
   - **Platform**: iOS
   - **Name**: `MAK — Beauty & Travel Buddy`
   - **Primary Language**: English (U.S.)
   - **Bundle ID**: select `com.mak.makeupbuddy` (from the dropdown — will appear after Phase A)
   - **SKU**: `MAK001`
   - **User Access**: Full Access
3. Click **Create**

---

### PHASE C: Build the IPA via Emergent

1. In Emergent dashboard, click **"Redeploy Changes"** (Replace Deployment + Use Existing DB)
2. Wait ~2-3 minutes for deployment
3. Click **"Get iOS Builds"** (sister button to "Get Android Builds")
4. When prompted, paste your **Team ID: `964BYV4G59`**
5. Verify the bundle ID shown is `com.mak.makeupbuddy`
6. Click **Generate New Build**
7. Wait ~10-15 min for the IPA to compile
8. Download the `.ipa` file when ready

---

### PHASE D: Upload IPA to App Store Connect

You can use either method:

**Method 1 (easiest — uses Transporter app):**
1. Download **Transporter** from the Mac App Store
2. Open Transporter → sign in with your Apple Developer Apple ID
3. Drag the `.ipa` file into the Transporter window
4. Click **Deliver**
5. Wait for processing — you'll see ✅ when it's accepted

**Method 2 (web upload):**
1. App Store Connect → My Apps → MAK → **TestFlight** tab
2. Click **+** next to **iOS Builds** → upload the .ipa
3. Wait for processing (~10-30 min — Apple sends an email when ready)

After processing, the build appears in TestFlight. Take ~30 minutes for export compliance — it auto-resolves.
